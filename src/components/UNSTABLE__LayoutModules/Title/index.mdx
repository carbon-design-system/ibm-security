# Title

## Overview

Titles provide interchangeable and reliable headings for establishing consistent hierarchies when using layout modules. Everything you pass in as a child will be rendered.

<Canvas>
  <Story id="patterns-unstable-layout-modules-title--default" />
</Canvas>

## Component API

<ArgsTable />

Additional props passed into `Title` will be forwarded along to the underlying title.

### `subsection`

You can use the `subsection` prop to specify whether a subsection should be used.

<Canvas withSource="none">
  <Default subsection>Subsection title</Default>
</Canvas>

```jsx
<Title subsection>Subsection title</Title>
```

## `element`

This prop allows you to specify a different element to be rendered as a title.

<Canvas withSource="none">
  <Default element="h3">Section 3 title</Default>
</Canvas>

```jsx
<Title element="h3">Section 3 title</Title>
```

### `className`

The `className` prop passed into `Title` will be forwarded along to the underlying title container. This is useful for specifying a custom class name for layout and [typography](#references).

```jsx
<Title className="bx--type-productive-heading-03">Custom title</Title>
```
